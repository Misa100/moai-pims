How to call it from N8N

Supabase → “Call a Postgres Function (RPC)”
Function: upsert_product_from_supplier
Body:

{ "p": {{$json}} }


(Assumes $json in N8N holds the supplier product object.)

Supabase SQL (Run Query) alternative:

select * from public.upsert_product_from_supplier(:p::jsonb);


with bind param p = {{$json}}.

2) Optional: set requested quantity
create or replace function public.set_product_quantity(p_product uuid, p_qty numeric)
returns void language sql as $$
  insert into public.product_prices(product_id, quantity)
  values ($1, coalesce($2,0))
  on conflict(product_id) do update
    set quantity = coalesce(excluded.quantity,0),
        updated_at = now();
$$;


N8N usage (after upsert returns product_id):
Call RPC set_product_quantity with { p_product: {{$json.product_id}}, p_qty: {{$json.quantity}} }.

3) Optional: bulk variant (array of objects)

If you ever fetch in bulk:

create or replace function public.upsert_products_from_supplier(p_list jsonb)
returns table(product_id uuid, was_insert boolean, was_update boolean)
language plpgsql
as $$
declare
  rec jsonb;
begin
  if jsonb_typeof(p_list) <> 'array' then
    raise exception 'p_list must be a JSON array';
  end if;

  for rec in select * from jsonb_array_elements(p_list)
  loop
    return query
      select * from public.upsert_product_from_supplier(rec);
  end loop;
end $$;

4) Notes / behavior

Existing SKU = update: Your product record’s primary key is the supplier product_id (UUID). If the product already exists, the products UPSERT hits the on conflict(id) path (update). If you also have a unique index on search_name (SKU), that prevents duplicate SKUs. Your N8N flow should still fetch by SKU to obtain the supplier JSON, then call this function.

Translations: English/French product texts are updated idempotently.

Categories: Uses your upsert_category_path('category', [...]) and also builds catalog from catalogGroup / catalogSubgroup.

Attributes: Creates color and material attribute options from the payload, with translations; links to the product.

Composition: First entry of articleMaterialComposition is applied (you can easily extend to loop all).

Prices: product_prices.price_supplier mirrors the latest exportPrice. duty_tax is set from intrastat_duty_tax when present, else 1. You can keep your triggers too—this UPSERT won’t fight them.
